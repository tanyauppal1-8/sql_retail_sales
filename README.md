# Retail Sales Analysis Using SQL

## About the Project

This project is a hands-on SQL analysis of a retail sales dataset. The goal was to practice writing SQL queries to clean data, explore business trends, and answer common business questions using real-world sales information.

The dataset contains transaction details such as customer information, product categories, quantity purchased, and sales amounts. Using MySQL, I performed data cleaning, exploratory analysis, and solved several business-focused problems to extract meaningful insights.

---

## What I Worked On

### Data Cleaning

Before starting the analysis, I cleaned the dataset by removing records with:
* Missing values
* Invalid sales information
* Missing customer details
* Missing product information

This ensured that the analysis was based on reliable data.

### Data Exploration

I explored the dataset to understand:

* Total number of sales transactions
* Number of unique customers
* Product categories available
* Overall structure of the data

---

## Business Questions Answered

Some of the questions I solved using SQL include:

* Retrieve all sales made on a specific date.
* Find Clothing category purchases where more than 3 items were sold during November 2022.
* Calculate total sales generated by each product category.
* Find the average age of customers purchasing Beauty products.
* Identify transactions with sales greater than 1000.
* Analyze transactions by gender and category.
* Determine the best-performing month in each year based on average sales.
* Find the top 5 customers by total spending.
* Count unique customers across different product categories.
* Analyze order volume across Morning, Afternoon, and Evening shifts.
